# HTTP Surface Contract: Coolify Docker Deploy

The runtime container exposes a single HTTP surface on internal port `80`. There is no JSON API; the contract is the set of routes nginx must answer.

## Routes

| Method | Path pattern | Response | Notes |
|---|---|---|---|
| `GET` | `/` | `200`, `Content-Type: text/html`, body = SPA shell (`index.html`), `Cache-Control: no-cache` | Always serves the latest deployed shell so visitors pick up new bundles after redeploy. |
| `GET` | `/assets/*` (and any other hashed asset emitted into `dist/assets/`) | `200`, correct MIME type, `Cache-Control: public, max-age=31536000, immutable` | Hashed filenames make aggressive caching safe. |
| `GET` | `/<existing-static-file>` (e.g. `/favicon.ico`, `/robots.txt`, `/manifest.webmanifest` if present) | `200`, correct MIME type | Served directly when the file exists in `dist/`. |
| `GET` | `/<arbitrary-client-route>` (any path that does not match a built file) | `200`, `Content-Type: text/html`, body = SPA shell, `Cache-Control: no-cache` | SPA fallback via `try_files $uri $uri/ /index.html`. Required for FR-006 and US2. |
| `GET` | `/health` | `200`, `Content-Type: text/plain`, body = `ok\n` | Generated by nginx via `return 200 "ok\n";`. No disk file. Required for FR-007 and US3. |
| Any other method on any path | `405` or `404` per nginx defaults | Acceptable | Out of scope; the SPA only needs `GET`. |

## Non-route guarantees

- **Port**: container listens only on `80`.
- **No env vars consumed**: nginx config is fully static.
- **No outbound calls**: the container does not connect to any external service.
- **No secrets in image layers**: enforced by `.dockerignore` and by the multi-stage build (build-stage filesystem is discarded).
- **MIME types**: rely on the default `mime.types` shipped with `nginx:alpine`. No overrides.

## Negative tests (must NOT happen)

- A request to `/some/deep/route` MUST NOT return `404`.
- A request to `/health` MUST NOT depend on disk I/O against the SPA bundle (so a missing file does not cascade into a failed healthcheck).
- A request for `index.html` MUST NOT be cached by intermediaries beyond the current request.
